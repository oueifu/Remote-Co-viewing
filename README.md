# Remote Co-viewing

一个轻量级的异地同步观影网页应用。它可以让两个人进入同一个房间，同步播放进度、发送聊天/弹幕、加载本地或网络视频，并通过 WebRTC 直接在网页内连麦通话。

项目不上传本地电影文件。服务端主要负责网页静态资源、WebSocket 房间同步、B 站弹幕代理和 WebRTC 信令转发。

## 功能总览

- 同步播放、暂停、拖动进度和播放速度
- 本地电影选择播放，双方只同步状态，不传输本地视频文件
- 网络视频直链加载，支持 B 站链接解析和历史弹幕加载
- Glassmorphism 毛玻璃影院界面
- 自定义浮动控制条，播放中鼠标静止后自动隐藏
- 全屏模式下右侧毛玻璃聊天抽屉，鼠标靠右或按 `C` 唤出
- 私人聊天消息同步显示为弹幕，本人紫色，对方蜜桃粉，B 站历史弹幕为灰色
- WebRTC 点对点音视频连麦，服务端只转发信令，不中转音视频流
- 连麦自动开启浏览器回声消除、降噪和自动增益
- 连麦时自动压低电影音量，挂断后恢复
- mpv 客户端模式，适合浏览器无法稳定播放的本地视频格式

## 目录结构

- `server.js`：HTTP 静态服务、WebSocket 房间同步、B 站弹幕/视频辅助接口、WebRTC 信令转发
- `public/index.html`：网页结构
- `public/styles.css`：毛玻璃影院界面样式
- `public/app.js`：浏览器播放器、房间同步、弹幕、聊天、连麦逻辑
- `mpv-client.js`：mpv 桌面播放同步客户端
- `package.json`：项目脚本和依赖

## 环境要求

- Node.js 18 或更高版本
- Chrome、Edge、Safari 等现代浏览器
- 如果要使用 mpv 模式，需要额外安装 `mpv`
- 远程使用连麦时，页面必须运行在 `https://` 或 `localhost` 下，否则浏览器通常会拒绝摄像头和麦克风权限

## 安装

```powershell
npm install
```

## 启动服务

默认端口是 `5050`：

```powershell
npm start
```

打开：

```text
http://localhost:5050
```

如果要换端口：

```powershell
$env:PORT="5051"; npm start
```

然后打开：

```text
http://localhost:5051
```

## 浏览器观影使用方法

1. 双方打开同一个服务地址，例如 `http://你的服务器地址:5050`。
2. 在右侧「房间与配置」里填写相同房间名。
3. 填写昵称。
4. 点击「连接」。
5. 一方加载视频或选择本地文件。
6. 点击播放后，播放、暂停、进度调整会同步到同房间成员。

## 邀请链接

右侧「复制」按钮会复制当前房间邀请链接。

对方打开邀请链接后，会自动带上房间名。若链接包含 `autoconnect=1` 或预设服务地址，也可以自动连接。

## 本地电影

点击「选择本地电影文件」选择电脑上的视频。

注意：

- 本地文件不会上传到服务端。
- 如果双方都看本地电影，最好选择同一个视频文件，片长和编码一致，同步效果最好。
- 浏览器对格式支持有限，`mp4`、`webm`、`mov` 通常更稳定。
- `mkv`、`rmvb`、H.265、复杂音轨等格式推荐用 mpv 模式。

## 网络视频

在「网络视频」输入框粘贴视频直链，然后点击「加载」。

建议使用浏览器能直接播放的地址，例如：

- `.mp4`
- `.webm`
- `.m3u8`

如果使用 B 站链接：

- 需要先连接房间，再点击加载。
- 服务端会尝试解析 B 站视频信息和弹幕 CID。
- 如果 B 站返回音视频分离的 DASH 流，浏览器直链可能只有画面没有声音，应用会提示改用 MP4/m3u8 直链或 mpv 模式。
- B 站历史弹幕会以灰色弹幕显示在视频上方。

## 播放控制

左下角按钮控制播放/暂停。

进度条支持点击跳转。

「对齐」按钮会手动拉齐到房间里最新同步状态，适合网络波动或刚加入房间后使用。

音量按钮和滑块只影响当前浏览器，不会同步给对方。

## 全屏和影院交互

点击「全屏」进入全屏影院模式。

播放中：

- 鼠标静止约 2.5 秒后，底部控制条会自动淡出。
- 鼠标指针会一起隐藏。
- 移动鼠标后控制条立即出现。
- 鼠标滑到屏幕右侧边缘会唤出聊天抽屉。
- 按 `C` 可以显示或隐藏右侧聊天抽屉。
- 按 `D` 可以显示或隐藏弹幕。

暂停时控制条会保持显示，方便调整进度和音量。

## 聊天和私人弹幕

右侧聊天室发送消息后：

- 消息会显示在聊天记录里。
- 同时会作为私人弹幕飘过视频。
- 自己发送的弹幕是紫色发光样式。
- 对方发送的弹幕是蜜桃粉发光样式。
- 系统提示不会作为弹幕发出。

## WebRTC 连麦

右侧顶部的连麦面板包含：

- 本地圆形视频头像
- 对方圆形视频头像
- 「连麦/挂断」按钮
- 麦克风开关
- 摄像头开关

使用方式：

1. 双方先连接同一个房间。
2. 任意一方点击「连麦」。
3. 浏览器请求摄像头和麦克风权限时点击允许。
4. 对方浏览器会自动接听并建立 WebRTC P2P 连接。
5. 点击「挂断」结束通话。

连麦特性：

- 音视频流不经过服务端，浏览器之间点对点传输。
- 服务端只转发 `rtc_offer`、`rtc_answer`、`rtc_ice`、`rtc_hangup` 这些信令消息。
- 麦克风默认开启浏览器内置回声消除、噪声抑制和自动增益。
- 连麦时电影音量会自动压低到 20%，减少回声和电影声干扰。
- 挂断后会恢复连麦前的电影音量。

重要限制：

- 摄像头/麦克风只能在 `localhost` 或 HTTPS 页面中正常使用。
- 如果远程通过普通 `http://服务器IP:5050` 打开，浏览器大概率会拒绝媒体权限。
- 远程连麦推荐使用 HTTPS 部署、Cloudflare Tunnel、ngrok、Tailscale Funnel 等方式暴露安全地址。
- 如果双方网络都在严格 NAT 后面，纯 STUN 可能无法打洞成功，这时需要 TURN 服务器。本项目默认不内置 TURN，保持轻量。

## mpv 模式

mpv 模式适合本地视频格式复杂、浏览器无法播放或音轨不兼容的情况。

先启动服务端：

```powershell
npm start
```

然后在客户端运行：

```powershell
npm run mpv -- --server ws://127.0.0.1:5050/ws --room movie-room --name 我的名字 --file "D:\Movies\movie.mkv"
```

远程示例：

```powershell
npm run mpv -- --server ws://你的服务器地址:5050/ws --room movie-room --name Windows --file "D:\Movies\movie.mkv"
```

macOS 示例：

```bash
npm run mpv -- --server ws://你的服务器地址:5050/ws --room movie-room --name Mac --file "/Users/me/Movies/movie.mkv"
```

## 远程部署建议

远程两端需要访问同一个服务地址。

常见方式：

- 云服务器部署并开放端口 `5050`
- 家中电脑端口转发
- Tailscale 或 ZeroTier 组网
- Cloudflare Tunnel
- ngrok
- 反向代理到 HTTPS 域名

如果需要 WebRTC 连麦，请优先选择 HTTPS 访问方式。

## 常见问题

### 浏览器阻止自动播放

先手动点击一次播放按钮。浏览器通常要求用户有一次手势操作后才允许带声音播放。

### 本地视频双方不同步

确认双方选择的是同一个文件，片长一致。不同版本、不同片头或不同编码文件会造成进度偏差。

### B 站视频有画面没声音

B 站经常返回音视频分离的 DASH 流。浏览器直接播放纯视频轨会没有声音。请改用 MP4/m3u8 直链，或使用 mpv 模式。

### 连麦打不开摄像头

确认页面地址是 `localhost` 或 HTTPS。普通 HTTP 远程地址通常无法使用摄像头和麦克风。

### 连麦有回声

推荐佩戴耳机。应用已经开启浏览器回声消除，并会在连麦时自动压低电影音量，但外放环境下仍可能有少量回声。

### 全屏控制条不隐藏

确认视频正在播放。暂停状态下控制条会保持显示。播放中鼠标静止约 2.5 秒后会自动隐藏。
